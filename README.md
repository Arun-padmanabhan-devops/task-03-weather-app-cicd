# 🚀 Task 03 - Weather App CI/CD Pipeline with Docker & GitHub Actions

## 📌 Project Overview

This project demonstrates a complete **CI/CD pipeline** for a containerized Weather Application using **Docker**, **GitHub Actions**, and **Docker Hub**.

Whenever code is pushed to the **main** branch, GitHub Actions automatically:

- Checks out the source code
- Builds a Docker image
- Logs in to Docker Hub
- Tags the image with:
  - `latest`
  - GitHub Actions Build Number
- Pushes both images to Docker Hub

This project simulates a real-world DevOps CI/CD workflow.

# ⚙️ GitHub Actions Workflow

The workflow is located at:

```
.github/workflows/docker-pipeline.yml
```

The pipeline executes the following stages:

1. Checkout Source Code
2. Login to Docker Hub
3. Build Docker Image
4. Tag Docker Image
5. Push Latest Image
6. Push Build Number Image

# 🏷️ Docker Image Tagging Strategy

Each successful pipeline generates two image tags.

Example

```
latest

1

2

3

4
```

Where

- `latest` → Most recent image
- `Build Number` → Unique image version generated by GitHub Actions (`github.run_number`)

This allows easy rollback to previous versions.
